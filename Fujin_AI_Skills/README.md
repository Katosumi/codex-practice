# Fujin AI ｜ 購入者向け使い方ガイド

『Codexアプリの教科書』特典④。
Fujinの**思考のクセ × 文体 × 判断軸**を移植した、Claude Agent Skills形式のクローンエージェントです。

---

## このSkillで何ができるか

ローカルのClaude Code／Codex／Claude Desktop に読み込ませることで、こんな使い方ができます：

- **AI時代のコンテンツ戦略・ローンチ戦略の壁打ち** をFujin思考でやってもらう
- **記事・X・LINE・メルマガ原稿のFujin風リライト**
- **発信ネタの言語化** （「これってFujinならどう切る？」のシミュレーション）
- 自分の原稿に対して「**Fujinならどう判断するか**」のセカンドオピニオン

> **重要：** これはFujinの代替ではなく、**Fujinの思考パターンを使って自分で考えるための補助輪** です。Skill自体に売上数字・顧客リスト・パートナー名などの個人情報は含まれていません。

---

## インストール手順

### Claude Code の場合

1. このフォルダ（`Fujin_AI_Skills`）をまるごと以下のパスにコピー：

```
~/.claude/skills/fujin-ai/
```

最終的にこういう構造になればOK：

```
~/.claude/skills/
└── fujin-ai/
    ├── SKILL.md
    ├── persona.md
    ├── thinking.md
    ├── style.md
    ├── examples.md
    └── README.md
```

2. Claude Code を再起動

3. プロンプトで「Fujin式で書いて」「Fujinならどう考える？」と話しかけると自動発動

### Codex アプリの場合

1. このフォルダを Codex アプリが読みに行ける場所に配置（例：`~/codex/skills/fujin-ai/`）
2. AGENTS.md にこの一文を追加：

```markdown
## Fujin AI Skill

Fujinの思考フレームと文体を再現したい場合は、`~/codex/skills/fujin-ai/SKILL.md` を読み込んで、その指示に従って振る舞ってください。
```

3. Codex アプリで「Fujin式で書いて」と依頼

### Claude Desktop / ChatGPT の場合

`SKILL.md` の中身を Project の指示文（System Prompt）に貼り付ければOK。
詳細が必要なときは `persona.md` `thinking.md` `style.md` `examples.md` の中身を追加で貼る。

### Notebook LM の場合

このフォルダの全ファイルを Source として追加してください。
Q&Aで「Fujinならどう判断する？」と聞くだけで、判断軸を踏まえた回答が返ってきます。

## 推奨の使い方パターン

### パターン1｜記事のFujin化

```
このドラフトをFujin式でリライトしてください。
（記事本文を貼る）
```

→ Skillが自動で `style.md` を参照して文体を調整、`persona.md` の口調に揃えます。

### パターン2｜判断の壁打ち

```
新商品のリリースタイミングをどう判断すべきか、Fujinの思考軸で壁打ちしてください。
状況：
- AI関連商品で、市場は今が旬
- 中身は7割完成
- 来月だと別の商品とぶつかる
```

→ Skillが `thinking.md` の7軸を当てはめて回答します。

### パターン3｜Xポストの量産

```
「Codex × MCP × AGENTS.md」のテーマで、Fujin式のXポストを5本作ってください。
```

→ Skillが `style.md` の掛け算×ステップ型を使って量産します。

### パターン4｜セカンドオピニオン

```
このローンチ戦略案、Fujin的に見て致命的に外している点はある？
（戦略案を貼る）
```

→ Skillが7軸でチェック、優先度の高い指摘から返します。

## 注意事項

### Skillに含まれていないもの

- Fujinの売上数字・顧客リスト・パートナー名・LINE登録者数などの**個人／事業情報**
- 特定の商品の価格・販売部数・契約条件
- Fujinの政治的・宗教的見解、特定個人への評価

これらをFujinが言ったことにして使うのはNGです。

### Skillの限界

- これは思考のクセを移植したもので、**Fujin本人ではありません**
- 重要な判断（高額契約、登壇、商品設計の最終決定）は本人に確認してください
- このSkillの回答を「Fujinの公式見解」として外部公開しないでください

### アップデート

- このSkillは Fujin の発信内容に合わせて随時アップデートされます
- 最新版は購入者専用LINEオープンチャットで配布

## トラブルシューティング

### Q. 「Fujinっぽくない」回答が返ってくる

→ プロンプトに「`SKILL.md` を読み込んでから回答してください」と明示的に書いてみてください。Claude側が他の指示を優先している可能性があります。

### Q. 同じ語尾（「じゃあね！」）が毎回出てきて違和感がある

→ Skillはあくまでテンプレートなので、毎回必ず使う必要はないです。「締めの定型は今回はナシで」と一言添えるだけで調整できます。

### Q. SKILL.md と persona.md の内容が矛盾しているように見える

→ `SKILL.md` は概要、`persona.md` は詳細です。詳細側（persona.md）を優先してください。
